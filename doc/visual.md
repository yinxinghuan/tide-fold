# Tide Fold 视觉圣经

## 1. 视觉命题

“把一整片海折进手机。”画面不是自然主义海景，而是原作实验室里一块具有重量、厚度和折射的深蓝流体。用户记住的不是 HUD，而是水体被两面看不见的墙压成中央巨浪。

## 2. 构图与相机

- 保留原作 MLS‑MPM 相机目标、45° FOV、斜俯视和 cubemap 空间关系；产品模式允许单指/鼠标旋转相机。
- 水体占竖屏中央 64–78% 高度，峰值仍保留完整外轮廓；上方 16% 与下方 13% 留给轻 UI。
- 390×844 为主构图，320×568 为短屏门禁；不套手机外框，不模拟设备或网页卡片。

## 3. 色彩系统

- `Ocean Mass #082E8C`：原流体主体，由原 shader 与 cubemap 产生，不以 CSS 重着色。
- `Abyss #07141F`：产品背景与错误页。
- `Foam Type #E8F3F4`：标题与主要动词。
- `Tide Line #79D7D2`：压力/折叠刻度与 ready 状态。
- `Muted Brine #7D969E`：实验编号和辅助信息。
- 禁止紫色霓虹、彩虹色散、粉色高光和额外渐变光晕。

## 4. 排版

- 标题：系统无衬线 `Arial Narrow, Helvetica Neue, Arial, PingFang SC`，600，16–20 px，全大写，字距 0.12em。
- 实验编号：同族 10–11 px，字距 0.18em。
- 当前动词：12–14 px，600，字距 0.14em；中文使用 `PingFang SC, Microsoft YaHei`。
- 不使用展示性衬线、液体字体或海报式巨大标题。

## 5. 形状、材质与光

- 主材质完全沿用原屏幕空间深度、厚度、双边滤波、Gaussian 和 cubemap 反射。
- UI 只有 1 px 直线、短刻度和无填充文字，不使用圆角卡、磨砂玻璃、投影或胶囊按钮。
- 幽灵手指为 Google Material `touch_app` 单色轮廓；不自行绘制卡通手掌。

## 6. 素材规则

- 使用上游 `public/cubemap/*.png` 六面图与全部 WGSL shader，不替换成生成图片或视频。
- 上游源码固定 `matsuoka-601/WebGPU-Ocean@3bd932778650b5e756ba2590969ed618313843ad`，许可证 MIT。
- 正式海报另走 Aigram transit，必须表现一块被无形力量折叠的深蓝实体海潮，不使用游戏截图。

## 7. UI 与图标

- 左上：`TIDE FOLD` + `STUDY 09 / MLS-MPM`。
- 两侧：各一条极细竖向边界线，折叠时向中央靠拢；它们是原箱体参数的可视化，不是假墙实体。
- 底部：只显示“双指捏合海潮”的隐藏手势提示，首次成功后淡出；折叠进度由两侧刻度与水体形变呈现。
- 错误状态：自定义线性刷新 SVG + 文字按钮，触控区域至少 44×44。

## 8. 动效与 VFX

- 输入同帧移动边界线并开始原物理压缩；UI 过渡 160–220 ms，物理不做 CSS 缓动。
- 不自动演示单指折叠；双指隐藏手势以一行文字交代，不占用原作自由相机的单指通道。
- 松手不加粒子、闪白或镜头震动；水体反冲本身是峰值。提示在 220 ms 内从 ready 切为 release，再于 1,650 ms 恢复。
- `prefers-reduced-motion` 下禁用幽灵往复，只显示静态起止轨迹 1.2 秒；真实物理仍响应用户。

## 9. 参考转译

- 直接保留：WebGPU Ocean 的 70,000 粒子 MLS‑MPM、固定参数、screen-space fluid rendering、cubemap 和默认相机。
- 只转译：原“Box width”滑杆变成全画布横向手势；原表单与调试文字在产品模式隐藏。
- 不借用：参考站点默认浏览器表单样式、Font Awesome GitHub 图标和桌面四角说明布局。

## 10. 反模式

- 禁止设备外框、玻璃卡片、巨型开始按钮、说明页、分数、倒计时和排行榜。
- 禁止自行添加泡沫粒子、海鸥、月亮、岛屿、闪电、碎片或发光拖尾。
- 禁止改成平面 Canvas 波纹、Three.js 网格海面或视频；WebGPU 不可用时诚实报错。
- 禁止同一手势同时旋转相机与折叠海潮；单指属于相机，双指属于折叠。

## 11. 垂直切片验收

- `?baseline=1` 在 Chrome 显示原 70,000 粒子 UI 与同一深蓝水体。
- 产品 390×844 首帧 3 秒内见到完整海潮、标题和真实效果幽灵引导。
- 拖动 42% 屏宽后边界靠拢，水体被明显压厚；松手后 2 秒内产生反冲并恢复可重玩。
- 320×568 下标题、动词与手指不裁切，不覆盖主体中心。
- WebGPU adapter 不可用时出现清晰错误与重试，不停在空白 loading 文案。

## 屏幕与状态合同

| 状态 | 玩家问题 | 视觉焦点 | 动作/恢复 |
|---|---|---|---|
| loading | 画面何时出现？ | 深色空场 + 一条细潮线 | GPU 首帧后自动进入 guide |
| guide / idle | 怎么碰它？ | 海潮 + 真实折叠幽灵演示 | 任意按下取消演示 |
| folding / ready | 我压了多少？ | 水体厚度和两侧边界 | 继续拖或松手 |
| release / recovery | 发生了什么？ | 原物理反冲与回落 | 1.65 秒后自动可重玩 |
| error / lost | 为什么没画面？ | 错误标题与一行原因 | 重新尝试刷新页面 |

## 反馈矩阵

| 输入 | 即时确认 | 峰值 | 恢复 |
|---|---|---|---|
| pointer down | 两侧线同帧点亮；70 ms 低音 | 无 | 进入 folding |
| horizontal drag | 边界与真实箱体同步内收；噪声音高随速度 | `fold≥0.72` 显示 RELEASE | 等待松手 |
| release | 动词立即切为 SURGE；420 ms 低频下滑 | 同一水体反冲 | 1.65 秒容器恢复，物理继续归静 |
| device lost | 停止循环；文字+图标 | 无 | 明确重试按钮 |
